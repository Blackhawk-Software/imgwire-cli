# Agent Notes

## Project Shape

This is a fresh oclif v4 TypeScript CLI scaffold for the `imgwire` binary.
It is ESM (`"type": "module"`), uses strict TypeScript, and expects Node
`>=18.0.0`.

Key paths:

- `src/index.ts` exports oclif's `run`.
- `src/commands/**` contains command implementations.
- `test/commands/**` contains command tests using `@oclif/test`.
- `bin/dev.js` runs the CLI from source during development.
- `bin/run.js` runs the built CLI.
- `README.md` has oclif-managed usage and command sections.

## Command Generation

Prefer the oclif generator when adding commands. Start with:

```sh
yarn oclif generate command <command-name>
```

Use nested names for topics/subcommands when appropriate, for example:

```sh
yarn oclif generate command images:upload
```

The generator writes into `src/commands` by default. If the command directory
ever changes, pass `--commands-dir <path>`. Avoid creating command files by hand
unless the generator cannot express the needed shape; if you do, match the
generated class/export conventions already in `src/commands`.

After adding or changing commands, update tests and regenerate oclif docs:

```sh
yarn oclif readme
```

For packaging-related changes, also verify the manifest path with:

```sh
yarn oclif manifest
```

Do not commit generated `dist/` or `oclif.manifest.json` files unless the repo
policy changes; both are ignored.

## Development Commands

- Install dependencies with `yarn`.
- Build with `yarn build`.
- Run tests with `yarn test`; `posttest` runs `yarn lint`.
- Run lint directly with `yarn lint`.
- Format files with `yarn format`.
- Check formatting with `yarn format:check`.
- Try commands from source with `./bin/dev.js <command>`.

The GitHub test workflow runs `yarn`, `yarn build`, and `yarn test` across
Ubuntu and Windows. Keep command behavior and path assumptions portable.

## Coding Conventions

- Keep code TypeScript-first and compatible with `module: Node16`.
- Use `@oclif/core` `Command`, `Args`, and `Flags` APIs for command surfaces.
- Put command parsing in `run()` via `await this.parse(CurrentCommandClass)`.
- Keep command output deterministic so tests can assert on `stdout`.
- Add tests near the command path under `test/commands`.
- Follow the generated style: no semicolons, compact static metadata, and oclif
  templating in examples where useful (`<%= config.bin %>`, `<%= command.id %>`).

## Documentation

The README command reference is generated by oclif. Edit prose outside the
`<!-- usage -->`, `<!-- commands -->`, and related generated marker blocks
manually; regenerate marked sections with `yarn oclif readme`.

When changing command descriptions, examples, args, or flags, update the command
class metadata first, then regenerate README output.

## Dependency Hygiene

This scaffold uses Yarn v1 and has a `yarn.lock`. Keep using Yarn for dependency
changes unless the project intentionally migrates package managers. Do not add
large frameworks or new runtime dependencies for simple CLI behavior; prefer
small helpers or existing oclif/core APIs.
